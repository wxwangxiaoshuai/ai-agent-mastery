## 什么是 Agent：从 Chatbot 到 Autonomous Agent

你已经会用 LLM 做问答（M1）、用 Prompt 做结构化输出（M2）、用 RAG 做知识库（M4）。但这些都是"你问一步，模型答一步"——模型不会主动搜索、不会自己决定下一步做什么。**Agent 的本质，是把"决定下一步做什么"的权力交给模型。**

### 从 Chatbot 到 Agent：四个概念的分界

```
LLM          →  给文本，预测下一个 token（纯推理引擎）
Chatbot      →  多轮对话，每轮独立回答（无状态、无行动）
Workflow     →  固定流程，LLM 在预设节点做处理（人决定流程）
Agent        →  自主循环，LLM 决定下一步做什么（模型决定流程）
```

**关键分界线**：Workflow 和 Agent 的区别在于**谁决定下一步**。

| 维度 | Workflow | Agent |
|------|----------|-------|
| 流程控制 | 开发者硬编码 | 模型动态决定 |
| 下一步 | 固定顺序 | 根据上一步结果推理 |
| 灵活性 | 低（改流程要改代码） | 高（模型自适应） |
| 可预测性 | 高（路径确定） | 低（路径可能变化） |
| 成本 | 低（只调必要的 LLM） | 高（每步都要调 LLM） |

**类比**：
- Workflow = **流水线工人**——按操作手册一步步做，不需要思考
- Agent = **独立贡献者**——给你一个目标，你自己决定怎么做

### Agent 的核心循环：感知 → 推理 → 行动

所有 Agent 的内核都是同一个循环：

```
while not done:
    1. 感知（Perceive）：获取当前状态（用户输入 + 工具返回 + 历史）
    2. 推理（Reason） ：LLM 思考下一步该做什么
    3. 行动（Act）    ：执行 LLM 决定的动作（调工具 / 返回答案）
    4. 观察（Observe）：获取行动结果，回到步骤 1
```

```python
# 最简 Agent 循环伪代码
def agent_loop(goal, tools, max_steps=10):
    messages = [{"role": "user", "content": goal}]
    for step in range(max_steps):
        # 1. 推理：LLM 决定下一步
        response = llm(messages, tools=tools)
        # 2. 行动：执行工具或返回答案
        if response.is_final_answer:
            return response.content
        tool_result = execute_tool(response.tool_call)
        # 3. 观察：把结果加入历史
        messages.append({"role": "assistant", "content": response.thought})
        messages.append({"role": "tool", "content": tool_result})
    return "达到最大步数，未能完成任务。"
```

这个循环就是 M5 全部的核心——后面 5 节课都是在讲**怎么让这个循环更智能、更可靠**。

### 自主性层级：L1-L5

不是所有"Agent"都一样自主。参照自动驾驶的分级，Agent 也可以分级：

| 层级 | 名称 | 特征 | 示例 |
|------|------|------|------|
| L1 | 辅助型 | LLM 只做建议，人执行 | 代码补全 |
| L2 | 部分自主 | 固定流程中，LLM 决定细节 | RAG 问答（检索→生成是固定的） |
| L3 | 条件自主 | LLM 决定步骤，但需人工审批 | HITL Agent（M10-03） |
| L4 | 高度自主 | LLM 全程自主，人在环外 | ReAct 研究 Agent（P5） |
| L5 | 完全自主 | LLM 自主设定目标、自我进化 | 目前尚不存在 |

**工程启示**：不要一上来就追求 L4/L5。大多数生产场景 L2-L3 就够了——L3 加人工审批往往比 L4 全自动更可靠、更安全。

### "Agent"这个词被滥用了

市场上什么都叫"Agent"——一个带 RAG 的聊天机器人也叫"Agent"。工程上，我们用一条简单规则判断：

> **如果去掉 LLM，系统还能完成同样的任务，那就不是 Agent。**

- RAG 问答：去掉 LLM，用关键词搜索 + 模板拼接也能做 → **不是 Agent**（是 Workflow）
- ReAct 研究 Agent：去掉 LLM，无法决定"搜什么、读什么" → **是 Agent**
- 多轮对话：去掉 LLM，无法回答 → 但也不算 Agent，因为它不"行动" → **是 Chatbot**

### Agent 的核心组件

```
┌─────────────────────────────────────────┐
│                Agent 内核                │
│                                         │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐ │
│  │  LLM     │  │  Tools  │  │ Memory  │ │
│  │ (推理引擎)│  │ (行动能力)│  │(状态记忆)│ │
│  └────┬────┘  └────┬────┘  └────┬────┘ │
│       │            │            │       │
│       └────────────┼────────────┘       │
│                    │                    │
│              ┌─────┴─────┐              │
│              │ Agent Loop │              │
│              │ (循环控制) │              │
│              └───────────┘              │
│                                         │
└─────────────────────────────────────────┘
```

- **LLM**：推理引擎——"下一步该做什么"
- **Tools**：行动能力——搜索、代码执行、数据库查询等
- **Memory**：状态记忆——对话历史、中间结果、已学经验
- **Agent Loop**：循环控制——什么时候停、什么时候继续、出错怎么办

> 这四个组件就是 M5-M10 的全部内容：M5 讲 Loop 和 LLM 推理、M6 讲 Tools、M7 讲可靠性、M8 讲 Memory、M10 讲框架编排。

### 要点总结

- Agent 的本质：把"决定下一步做什么"的权力从开发者交给模型
- 核心循环：感知 → 推理 → 行动 → 观察（无限循环直到完成）
- 自主性分级 L1-L5：生产场景推荐 L2-L3（带人工审批比全自动更可靠）
- 判断规则：去掉 LLM 还能完成同样任务的，不是 Agent
- Agent 四组件：LLM（推理）+ Tools（行动）+ Memory（记忆）+ Loop（控制）
- 后续 5 节课：手写 Loop（L05-02）→ ReAct 深入（L05-03）→ Plan-Execute（L05-04）→ Reflection（L05-05）→ 决策框架（L05-06）
