## 什么是大语言模型

大语言模型（Large Language Model，LLM）本质上是一个**超大规模的"下一个 token 预测器"**。你给它一段文本，它预测下一个最可能出现的 token，然后把预测结果拼回去，继续预测下一个——如此循环，直到生成完整的回复。

### Token 是什么

Token 是 LLM 处理文本的最小单位。它不是"字"也不是"单词"，而是介于两者之间的"子词"（subword）。

一个粗略的经验法则：
- 英文：1 token ≈ 0.75 个单词 ≈ 4 个字符
- 中文：1 个汉字 ≈ 1–2 tokens（通常比英文更「贵」）

### 概率采样：LLM 的"决策"方式

LLM 在每一步生成时，并不是只给出一个确定的 token，而是为词表中的每个 token 计算一个**概率**。

```
输入："今天天气真"
模型输出概率分布：
  好 → 0.65
  棒 → 0.20
  热 → 0.08
  冷 → 0.04
  蓝 → 0.01
  ...（其余 token 分享剩余概率）
```

默认情况下模型选概率最高的 token（"好"），但通过调整采样参数（temperature、top_p 等，下节课详解），你可以让它从概率分布中"抽样"，这就是 LLM "创造力"的来源。

**工程含义**：同一个输入每次调用可能得到不同输出——这对测试和可复现性是挑战，也是 L01-02 采样参数存在的意义。

### 上下文窗口

上下文窗口（Context Window）是模型一次能"看到"的最大 token 数量。它决定了：
- 你能塞多少背景信息
- 对话能持续多长
- Agent 能记住多少历史步骤

主流模型的上下文窗口（参考值，请以官方文档为准）：
- Claude 系列：200K tokens
- GPT 系列：128K tokens
- Gemini 系列：1M+ tokens

**工程约束**：窗口越大，推理越慢、成本越高。不要因为"窗口够大"就无脑塞内容——长上下文中的信息检索准确率会随位置变化（"Lost in the Middle"效应）。

### 为什么 LLM 会"幻觉"

幻觉（Hallucination）的根源在于 LLM 的工作方式——它不是在"查找事实"，而是在"预测概率最高的下一个 token"。

```
用户：巴黎是哪个国家的首都？
模型：法（概率最高）→ 国（概率最高）→ 。（概率最高）
```

这个过程中，模型没有"查数据库"，它只是根据训练数据中的统计模式来生成。当它遇到训练数据中信息不足或相互矛盾的问题时，就会"编造"一个看起来合理但实际错误的答案。

**降低幻觉的工程手段**：
- RAG（检索增强生成）：先查到真实信息，再让模型基于信息回答
- Function Calling：让模型调用工具获取实时数据
- 结构化输出：约束输出格式，减少自由发挥空间

### 涌现能力：量变引发质变

当模型规模（参数量 + 训练数据）跨越某个阈值后，会出现小模型不具备的能力——这就是**涌现能力（Emergent Abilities）**。就像水到 100°C 突然沸腾，你很难从 50°C 的水预测出"沸腾"这个现象。

已观察到的典型涌现能力：

| 能力 | 含义 | 对 Agent 开发的意义 |
|------|------|---------------------|
| In-Context Learning | 不用微调，仅靠 Prompt 中的示例就能学会新任务 | Few-shot Prompt 的基础 |
| Chain-of-Thought | 能"一步步推理"而非直接猜答案 | ReAct、Plan-Execute 范式的前提 |
| Instruction Following | 能理解并遵循复杂指令 | System Prompt 架构的基础 |
| Tool Use | 能选择并调用外部工具 | 整个 Agent 工具链的前提 |

**工程启示**：涌现能力不是"魔法"，它有边界——在模型不擅长的领域（如精确数学计算、实时信息），涌现能力会失效，这时必须用工具（Function Calling）来补偿。

### 训练、微调、推理三个阶段

```
预训练（Pre-training）    →  微调（Fine-tuning）    →  推理（Inference）
海量文本 → 基础模型        标注数据 → 对齐模型        用户输入 → 实时输出
数十亿美元                数千到数百万美元            按 token 计费
```

大多数 Agent 开发者只需要关注**推理**阶段——如何用好已有的模型。但理解前两个阶段能帮你更好地理解模型的"性格"和"边界"。

补充：现代 LLM 的对齐通常经过两步——**SFT**（Supervised Fine-Tuning，用人工标注的"指令-回复"对训练）和 **RLHF**（Reinforcement Learning from Human Feedback，用人类偏好做强化学习）。你不需要自己跑这些流程，但理解它们能解释为什么模型会"拒绝"某些请求、为什么会"过度礼貌"。

### 要点总结

- LLM 的本质是 token 预测器，不是数据库
- 每一步生成都产生概率分布，采样参数控制如何从分布中选取 token
- Token 是子词单位：英文约 4 字符/token，中文通常 1 汉字要 1–2 tokens
- 上下文窗口是 Agent 记忆的硬上限，且存在"Lost in the Middle"效应
- 幻觉来自概率预测机制，需要通过工程手段抑制
- 涌现能力（ICL、CoT、Tool Use）是 Agent 开发可行性的基础
- 作为开发者，你主要关注推理阶段，但理解训练和微调有助于模型选型
